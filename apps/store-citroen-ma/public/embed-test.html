How this differs from embed.js

The embed.js loader injects an iframe at /embed/[brand] and resizes it on open / close via postMessage. The plain <iframe> below points at /w/[brand] and uses the host CSS for sizing — better for page-builder slots (AEM, Salesforce, Webflow), worse for "drop on any site" use because it can't shrink to a FAB.
